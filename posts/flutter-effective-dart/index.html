[Flutter] Effective Dart (Part I)

Hades Dec 24, 2020 2020-12-24T14:15:03+00:00
6 min

Bài viết giúp bạn làm thế nào để xây dựng mã code clean clear hơn. Chi tiết ở đây

Để áp dụng vào dự án nhanh và dễ dàng hơn, dart analyzer cung cấp lint - một bộ quy tắc cần tuân theo khi viết code. Hãy tham khảo nó ở đây

Linting là một công cụ cực kì hữu ích. Nó giúp bạn thay đổi hành vi về cách viết code và giúp bạn code một cách clean.

Introduction

Bài viết bao gồm 4 phần :

  • Style Guide – Một số quy tắc bố trí và sắp xếp code, hoặc ít nhất là những gì mà dart format chưa handle cho bạn. Guide này cũng chỉ định cách format như nào cho hợp lý : camelCase, using_underscores, etc…

  • Documentation Guide – Hướng dẫn cách comment, viết docs…

  • Usage Guide – Làm cách nào để implement 1 đoạn mã sử dụng các statement hay expression hợp lý…

  • Design Guide – Đây là phần nhỏ nhưng scope rộng nhất. Cách khai báo, thiết kế consistent, usable APIs

Để xem 1 cách tổng quan, bạn có thể tham khảo đây.

Trong bài viết này, mình sẽ chia sẻ về 2 chủ đề đầu tiên nhé.

1. STYLE GUIDE

Một số quy tắc bố trí và sắp xếp code, hoặc ít nhất là những gì mà dart format chưa handle cho bạn. Guide này cũng chỉ định cách format như nào cho hợp lý : camelCase, using_underscores, etc…

1.1.1 UpperCamelCase

  • Viết hoa chữ cái đầu tiên của mỗi từ
  • Sử dụng với : Classes, enum types, typedefs, type parameters…

  • Tên viết hoa chữ cái đầu tiên của mỗi từ, ngoại trừ từ đầu tiên

    • class members, top-level definitions, variables, parameters, and named parameters
    • const variables

1.1.4 PREFER using _, __, etc. for unused callback parameters.

  • Khi callback params không dùng đến, nên dùng gạch chân _ . Giúp việc đọc hiểu nhanh hơn.

  • Sử dụng đường dẫn tương đối khi import

  • Export và import nên phân cách nhau
